# Briefly  
Инструмент для глубокого анализа требований и технических заданий

## Проблема  
Большая часть ТЗ в реальных проектах пишется «как получится».  
Формулировки расплывчатые, куски логики противоречат друг другу, половина решений подразумевается, а не прописана. В итоге команда тратит недели на уточнения, вылазят скрытые зависимости, а разработчики постоянно сталкиваются с “а что автор имел в виду?”.

Главная боль: **Copilot и подобные инструменты здесь бесполезны**.  
Они генерят код и локальные подсказки, но не понимают структуру требований, не ловят противоречия и не способны оценить зрелость ТЗ целиком.  
А человеческий ревью требований — дорого, долго и субъективно.

## Как Briefly решает проблему  
Briefly — это ИИ-анализатор требований, который прогоняет документ через набор интеллектуальных проверок и выдаёт структурированный отчёт. Сервис выявляет:

- неясные, размазанные или двусмысленные формулировки;  
- логические противоречия;  
- скрытые допущения;  
- пропущенные детали, без которых фича не будет реализуема;  
- потенциальные риски и зависимости;  
- вопросы, которые необходимо уточнить у постановщика.

Цель — дать команде **чёткое, проверяемое и приземлённое понимание задачи**, прежде чем кто-то начнёт писать код.

## Что входит в MVP Briefly
Минимальная рабочая версия, которую можно развернуть за пару недель, выглядит так:

### 1. Веб-интерфейс
- Поле для ввода/вставки текста ТЗ   
- Подска

### 2. API-backend
Единая точка входа `/analyze`, которая принимает текст и возвращает JSON с результатами проверки:
```json
{
  "issues" : [
    {...}
  ]
}
```

### 3. Интеллектуальный разбор ТЗ

На первом этапе анализ основан на LLM, но с жёсткой структурой промптов и форматом ответа, чтобы:

* выдавать стабильный JSON,
* не уходить в воду,
* обеспечить читаемость и ценность результата.

## Технологии

* Frontend: React + TailwindCSS
* Backend: Rust (Actix-web)
* LLM: Remote API (OpenAI, Anthropic и т.д.)
* База данных: SQLite/PostgreSQL (по необходимости, если будет решено хранить аккаунт и историю)
